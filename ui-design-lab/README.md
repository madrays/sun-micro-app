# UI Design Lab

用于微应用 UI 联调的辅助工具，目标是让「看图说话」变成「元素级可复制信息协作」。

## 能力

- 自动扫描仓库内微应用（基于 `config/components.config.js`）
- 自动提取 Widget 尺寸、`baseStyle`、`sizeStyle`（若存在）
- 按统一网格标准计算尺寸（默认 `unit=76`、`gap=22`）
- 支持 `1xfull` 全宽独占一行预览
- 预览支持深浅背景、文字颜色、昼夜模式
- 支持网格线与元素框线开关
- 点击任意元素复制 JSON（路径、尺寸、关键样式）
- 左侧一键 `刷新最新改动`，前端直接重建清单，不用手动跑生成命令
- 每个预览卡右上角放大镜循环缩放（100%/125%/150%/200%）
- 放大后自动切换单尺寸模式；切换全尺寸自动恢复标准预览
- 单尺寸或放大模式下支持编辑：拖动元素、`+/-` 缩放、`Shift +/-` 调整字体
- `复制更改` 仅输出最终 `before/after` 差异，便于直接发给 AI

## 启动

```bash
cd ui-design-lab
npm install
npm run dev
```

默认端口：`5176`

## 日常调试流程

1. 先选择微应用、Widget 和目标尺寸。
2. 修改真实组件代码后，点左侧 `刷新最新改动`。
3. 在右侧预览点击元素，复制结构和样式 JSON 发给 AI。
4. 若要微调布局，切到单尺寸（或先放大），进入编辑模式后调整。
5. 完成后点 `复制更改`，把最终差异发给 AI 落地代码。

## 编辑模式说明

- 入口：预览卡工具区 `编辑` 按钮（仅单尺寸或放大模式可用）
- 鼠标：点击元素选中，拖动可平移
- 键盘：`+/-` 缩放选中元素；`Shift +/-` 调整字体
- 导出：`复制更改` 会复制最终差异，不记录中间每一步
- 退出：再次点 `退出编辑`，或切换全尺寸模式

## 构建

```bash
npm run build
```

## manifest 生成机制

- `npm run generate` 会生成：

- `public/manifest.json`

该文件包含扫描出的微应用与组件设计信息，供预览端读取。  
开发时也可在前端点 `刷新最新改动`，其内部会调用 `POST /__manifest/refresh` 即时重建 `manifest.json`。

时间字段说明：

- `generatedAt`：上海时区格式（`Asia/Shanghai`）
- 页面展示：`版本时间：YYYY-M-D HH:mm:ss`

## 使用建议

1. 先用单尺寸定位问题尺寸（如 `2x2`）。
2. 开启网格线检查对齐与边距。
3. 点击异常元素复制 JSON，直接发给 AI。
4. 需要精调时进入编辑模式，完成后复制更改差异。
5. 修复后切换全尺寸做回归验证。

## 说明

如果某组件采用 `render1x1()/render2x2()` 这类函数内联 `<style>`，可能不会提取到独立 `sizeStyle` 对象；此时以真实组件渲染结果为准。
